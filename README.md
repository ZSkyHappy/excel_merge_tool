# Excel 批量合并工具 v2.0.0

面向非技术用户的 Windows 图形化表格合并工具。支持 `.xlsx`、`.xlsm` 和 `.csv`，先执行精确预检，再以流式方式生成结果与 Excel 处理报告，不会修改源文件。

## 最简单的使用方法

1. 双击 `ExcelMergeTool-Windows-x64.exe`。
2. 首次使用保持“经典 A:Q 模式”，选择存放源文件的文件夹和结果保存位置。
3. 点击“精确扫描”，在文件预览中检查有效、跳过和排除状态。
4. 如需调整某个文件的工作表、表头行、CSV 编码或分隔符，选中文件后点击“逐文件设置”。
5. 点击“开始合并”。完成后可直接打开结果或 `合并报告_*.xlsx`。

经典模式保持 v1 行为：读取每个文件的第一个可见工作表，保留 A:Q，在末尾追加“所属表格+主讲老师”，内容格式为 `文件名 - 老师`。

## 自定义字段映射方案

点击“新建”，选择一个 Excel、XLSM 或 CSV 作为字段模板。自定义方案支持：

- 选择、重排和重命名任意输出字段。
- 为字段配置多个别名，例如让“编号”同时匹配“ID”和“序号”。
- 将字段标记为必填或可选：缺少必填字段时跳过文件，缺少可选字段时补空。
- 保存多个本机方案，并通过 JSON 导入、导出给同事复用；方案文件不包含本地路径或业务数据。
- 可选扫描子文件夹；结果顺序按相对路径稳定排序。

同一源文件中出现规范化后重复的表头时，程序不会猜测映射，而是跳过并在预览与报告中说明原因。

## 特殊字段与标签列

特殊字段用于提取每个文件共有的标签信息，例如“主讲老师”“部门”“课程”。启用后，每个配置的特殊字段在单个文件中必须存在，而且只能有一个唯一非空值，否则该文件会被跳过。

“识别特殊字段”和“追加标签列”可以分别开关。标签模板可组合：

- `{文件名}`
- `{文件名不含扩展名}`
- `{相对路径}`
- `{工作表}`
- `{特殊字段:字段名}`

标签列始终追加在自定义输出字段末尾。

## 输入格式与数据规则

### Excel 和 XLSM

- 默认读取第一个可见工作表，可在预览中逐文件改选。
- 保留文本、数字、布尔值、日期和公式上次保存的缓存结果。
- 不执行或复制宏，不保留公式本身、外部链接、源样式和合并单元格。
- 如果公式没有由 Excel 计算并保存过，缓存结果可能为空。

### CSV

- 自动尝试 UTF-8 BOM、UTF-8 和 GB18030 编码。
- 自动识别逗号、制表符、分号和竖线；无法可靠识别时，可在逐文件设置中手动选择。
- 所有非空字段按原文本输出，避免编号或身份证号的前导零丢失。
- 支持标准 CSV 引号和字段内换行。

### 通用规则

- 映射后的数据字段全空时不输出该行；标签值不会让空数据行被保留。
- 非法 XML 控制字符会被移除，超过 Excel 单元格上限的文本会截断，并计入报告。
- 每个结果最多包含 1,048,575 条数据加一行表头；超限自动生成 `_第1部分`、`_第2部分` 等分卷。
- 输出统一冻结首行、启用筛选并按预检结果设置合理列宽。

## 精确预检与报告

精确预检会完整读取一次源文件，以确认：

- 表头和字段映射；
- 工作表、表头行、CSV 编码与分隔符；
- 数据行数；
- 特殊字段是否唯一；
- 文件是否损坏或缺少必填字段。

正式合并时再流式读取一次。文件在预检后发生变化时会被跳过，要求重新扫描。取消或致命失败时只清理本任务临时文件，不发布半成品。

处理报告为独立的 `合并报告_时间戳.xlsx`，包含：

- `任务汇总`：方案、路径、文件数、总行数、耗时和输出文件；
- `文件明细`：每个文件的状态、解析参数、特殊字段、写入行数和异常原因；
- `字段映射`：输出字段、必填规则、别名、特殊字段和标签模板。

程序崩溃时另行生成 `.log` 技术日志。

## 从源码运行与构建

项目固定依赖：

- Python 3.14.4
- openpyxl 3.1.5
- lxml 6.1.1
- PyInstaller 6.21.0（仅构建 EXE）

构建脚本只在项目内创建 `.venv`，但首次运行会从网络下载依赖并占用磁盘空间：

```powershell
.\build.ps1
```

脚本依次安装固定版本依赖、运行全部自动化测试、生成应用图标和 Windows 版本信息，并输出：

```text
dist\ExcelMergeTool-Windows-x64.exe
dist\SHA256SUMS.txt
```

重新构建并覆盖现有 EXE：

```powershell
.\build.ps1 -Force
```

从源码运行：

```powershell
.\.venv\Scripts\python.exe .\excel_merge_tool.pyw
```

## 测试与性能验收

运行63项自动化测试：

```powershell
.\.venv\Scripts\python.exe -m unittest discover -s tests -v
```

运行默认的10万行 × 17列经典 XLSX 性能测试：

```powershell
.\.venv\Scripts\python.exe .\benchmark.py
```

运行自定义映射 CSV 性能测试：

```powershell
.\.venv\Scripts\python.exe .\benchmark.py --format csv --profile custom --rows 100000
```

比较20万行时的峰值内存：

```powershell
.\.venv\Scripts\python.exe .\benchmark.py --rows 200000
```

性能会受硬盘、CPU、文本重复度和 Excel 压缩率影响，因此不承诺固定秒数。当前自动化测试使用合成数据；未提供实际业务样本，其兼容性标记为“未验证”。Windows 11 可在本机验收，Windows 10 在没有对应环境时标记为“未验证”。

## 暂不支持

- 旧版 `.xls`。
- 保留宏、公式本身、源样式或合并单元格。
- 去重、排序、汇总、数据计算或自动合并全部工作表。
- 安装程序、自动更新和数字签名。
