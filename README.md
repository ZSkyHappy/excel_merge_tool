# Excel 批量合并工具

这是一个面向非技术用户的 Windows 图形工具。它会合并指定文件夹第一层的 `.xlsx` 文件，保留 A:Q，并在 R 列写入来源文件名与主讲老师。

## 最简单的使用方法

1. 双击 `Excel批量合并工具.exe`。
2. 选择存放源 Excel 的文件夹。
3. 选择结果保存位置；默认输出文件名带时间戳，不会轻易与旧结果重名。
4. 点击“开始合并”，完成后点击“打开输出文件夹”。
5. 如果有文件被跳过，请查看同目录下的 `合并报告_*.txt`。

工具只扫描所选文件夹第一层，不扫描子文件夹；会自动忽略 `~$` 开头的 Excel 临时文件。源文件不会被修改。

## 固定合并规则

- 仅支持 `.xlsx`。
- 每个文件读取第一个可见工作表，第一行必须是表头。
- 首个有效文件的 A:Q 作为标准表头；其他文件比较时会去除表头首尾空格。
- 在首行查找唯一的“主讲老师”列，并读取数据区第一个非空值。
- 输出 A:Q 原始值，R 列标题为“所属表格+主讲老师”，内容格式为 `文件名 - 老师`。
- A:Q 全空的行不会输出；不复制公式和源单元格样式。
- 每个输出文件最多包含 1,048,575 条数据。超过后自动生成 `_第1部分`、`_第2部分` 等分卷。
- 输出采用统一表头样式、冻结首行和自动筛选。

损坏文件、缺少老师、老师为空、没有有效数据或表头不一致时，该文件会被跳过并写入报告。工具会先完整预检，再合并到任务临时目录；取消或失败时不会发布半成品。

## 从源码运行

本项目固定使用：

- Python 3.14.4
- openpyxl 3.1.5
- lxml 6.1.1
- PyInstaller 6.21.0（仅用于构建 EXE）

依赖安装会下载第三方包并占用磁盘空间。`build.ps1` 只在项目内创建 `.venv`，不会修改系统 Python：

```powershell
.\build.ps1
```

脚本会依次安装固定版本依赖、运行全部自动化测试，并生成：

```text
dist\Excel批量合并工具.exe
```

如果只想运行源码：

```powershell
.\.venv\Scripts\python.exe .\excel_merge_tool.pyw
```

## 测试与性能验收

运行自动化测试：

```powershell
.\.venv\Scripts\python.exe -m unittest discover -s tests -v
```

运行默认的 100,000 行 × 17 列性能测试：

```powershell
.\.venv\Scripts\python.exe .\benchmark.py
```

性能受硬盘、CPU、文本重复度和 Excel 压缩率影响，因此工具不承诺固定秒数。实际业务文件尚未提供，其兼容性为“未验证”；建议首次使用时先复制少量代表性文件试运行并核对报告。
